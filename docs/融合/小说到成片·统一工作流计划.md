# 小说到成片 · 统一工作流计划（固定标准）

> 本文档为 MYStudio（漫影工作室）从小说到成片的**唯一标准工作流**，融合 ToonFlow（编剧深度）、huobao-drama（提取/分镜/音色）、moyin-creator（制作/剪辑）三者之长。后续所有相关开发以本文为准。

---

## 0. 一句话结论

单看任一项目都不是最强：**huobao 编剧太浅、moyin 没编剧、ToonFlow 没制作/剪辑**。
最强方案 = **ToonFlow 的编剧深度（可调深浅）＋ huobao 的提取/分镜/音色 ＋ moyin 的制作/剪辑**，并用三个粘合点把它们焊成一体：**统一实体标签、剧集圣经、可调深度**。

---

## 1. 完整工作流（标准阶段）

输入：小说正文 → 最终产出：成片视频。分镜与剧本均为中间产物。

| # | 阶段 | 目的（解决什么落差） | 主产出 | 能力来源 |
|---|------|----------------------|--------|----------|
| 0 | 风格与导演选择 | 全程视觉/镜头基调先定 | visualManualId + directorManualId | moyin 风格库 + ToonFlow 手册 |
| 1 | 小说导入 | 正文按章入库 | NovelChapter[] | moyin/MYStudio |
| 2 | 事件分析 | 长正文→结构化"每章发生了什么"+重要度/时长 | 每章事件表（7字段）| ToonFlow |
| 3 | 故事骨架 | 定核心吸引力/主角弧/三幕/分集/付费卡点 | 骨架文档 | ToonFlow |
| 4 | 改编策略 | 小说"讲述"→影视"展示"的转换规则 | 改编策略 | ToonFlow |
| 5 | 细纲（可选）| 每集→每场 beat，降"骨架→剧本"跨度 | 分集细纲 | 新增（ToonFlow 增强）|
| 6 | 剧本 | 产出可拍文本，**必须为统一剧本格式** | 各集剧本 | ToonFlow 写作 + huobao 格式 |
| 7 | 实体提取 | 角色/场景/道具**去重**并按集关联 | 实体库 + 实体ID | huobao extractor |
| 8 | 分镜拆解 | 每场切成可生成镜头（含分段视频提示词）| 分镜表（标准字段）| huobao storyboard_breaker |
| 9 | 一致性资产 | 角色/场景定版（宫格图/参考图）| 资产图 + 锚点 | huobao grid + moyin 库 |
| 10 | 音色分配 | 角色→音色（性别/年龄/性格）| 角色音色映射 | huobao voice_assigner + 本地TTS |
| 11 | 生图 / 生视频 | 文字镜头→画面（首尾帧→图生视频）| 每镜片段 | moyin 导演/S级 |
| 12 | 剪辑成片 | 拼接+字幕+配音+BGM，本地合成 | 成片视频 | moyin + 本地 FFmpeg |

> 前置（阶段 0）的风格/导演手册作为阶段 3–12 的全局约束注入。

## 2. 三大粘合点（决定"是否真的最强"的关键）

### 2.1 统一实体标签（一致性骨架）
- 采用 huobao `storyboard_breaker` 的标签体系：`<location>地点</location>`、`<role>角色</role>`、`<voice>旁白/画外音</voice>`、`<n>` 时间段分隔。
- **从剧本→实体提取→分镜→生成**全程引用**同一套实体 ID**（characterId / sceneId / propId）。
- 效果：换角色/场景只改一处；同一角色跨镜头、跨集视觉一致；视频提示词可被生成端可靠解析。

### 2.2 剧集圣经（Series Bible，全局强制注入）
- 一个项目级"圣经"：锁定**角色外貌/音色、场景、画风(visualManual)、导演手册(directorManual)、画幅、风格定位**。
- 在阶段 3–12 的**每一次 Agent 调用/生成请求**头部自动注入，保证规模化（几十集）下不漂移。
- 现状：moyin 有局部库、MYStudio 有 `seriesMeta`/`workflowConfig`，但**未做全程强制注入**——本计划补齐。

### 2.3 可调深度（避免逼用户走全部步骤）
- 由项目配置（集数/单集时长）自动决定编剧链路深浅：
  - **短/简单**（如 1 集、3 分钟）：跳过事件/骨架/改编，走 huobao 式"小说→剧本"一步改写。
  - **长/复杂**（多集、需分集压缩/主线管理）：走 ToonFlow 深链（事件→骨架→改编→细纲→剧本）。
- 阶段 7–12（提取/分镜/生成/剪辑）无论深浅都一致。

## 3. 标准数据契约

### 3.1 统一剧本格式（阶段 6 产出，阶段 7/8 消费）
```
## S01 | 内景 · 咖啡厅 | 黄昏
环境描写……
角色动作描写……
小红：（微笑）等很久了吗？
```
- 不含镜头语言（景别/运镜留给分镜阶段）。每场 30–60 秒。

### 3.2 标准分镜字段（阶段 8，固定）
`title / shot_type / angle / movement / location / time / character_ids / action / dialogue / description / result / atmosphere / image_prompt / video_prompt / bgm_prompt / sound_effect / duration / scene_id`
- `video_prompt` 必须按 **3 秒分段**并使用统一实体标签。
- `scene_id` / `character_ids` 必须引用实体库中已存在的 ID（不得凭空生成）。

### 3.3 事件表字段（阶段 2，固定）
`章节 | 涉及角色 | 核心事件 | 主线关系 | 信息密度 | 预估集长(秒) | 情绪强度`（与现有 `event-analysis.ts` 一致）。

## 4. 技术映射（落到 MYStudio 现有结构）

- **AgentWorkKey**（types/studio.ts）已含：eventAnalysis / storySkeleton / adaptationStrategy / scriptDraft / productionPlan / directorPlan / deriveAssets / generateAssets / storyboardTable / storyboardPanel / storyboardImage / supervisionReport。
  - 新增：`episodeOutline`（细纲，置于 adaptationStrategy 与 scriptDraft 之间）、`entityExtraction`（实体提取去重）。
- **Skill 文档**：放入 `apps/frontend/assets/studio-manuals/`，复用 ToonFlow 的 `script_execution_*`、`production_execution_*`，新增 huobao 风格的 `entity_extractor.md`、`storyboard_breaker.md`、`voice_assigner.md`、`grid_prompt.md`（按本文契约改写为 MYStudio 版）。
- **Agent 绑定**：沿用 `api-config-store` 的 `AgentDeploymentKey`，每阶段可单独绑模型。
- **持久化**：工作流阶段与配置随项目存于 `studio-store`（`createProjectScopedStorage`，已支持）。
- **制作/剪辑**：沿用 moyin 导演板块 / S级 / Electron 主进程 FFmpeg。

---

## 5. 落地里程碑

- **M1 实体提取（最高性价比，当前最缺）**：移植 huobao `extractor` → 新增 `entityExtraction` 阶段（角色/场景/道具去重 + 按集关联），写入资产库。
- **M2 分镜标准化**：把 §3.2 分镜字段（含 3 秒分段 video_prompt + 实体标签）定为 MYStudio 分镜唯一标准，改造分镜表。
- **M3 剧集圣经**：在 `seriesMeta/workflowConfig` 上扩出 Series Bible，做"每次生成请求自动注入"。
- **M4 可调深度**：按项目配置切换"扁平改写 / 深链编剧"。
- **M5 细纲阶段**：新增 `episodeOutline`，剧本 Agent 改为消费细纲。
- **M6 音色闭环**：huobao `voice_assigner` 规则 + 本地 TTS 声音克隆贯通到分镜配音。

---

## 6. 验收标准

- 给定一本小说，可在一个 app 内**本地闭环**产出成片。
- 同一角色/场景跨镜头、跨集**视觉与音色一致**（实体ID + 圣经注入生效）。
- 短篇走扁平链 ≤ 3 步出剧本；长篇走深链可分集、可控压缩比。
- 每个中间阶段都有**人工可审可改**入口；任一步出错可回退重跑而不影响其它阶段。

---

> 维护：本工作流为固定标准。任何阶段的增删或字段变更，需更新本文件并同步 `types/studio.ts` 的 `AgentWorkKey` 与 `studio-manuals` 下对应 skill。
