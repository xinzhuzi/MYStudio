# Design

## Problem Restatement

MYStudio chapter-001 currently writes a short four-scene director plan, while Toonflow's real EP01 writes a long six-section, five-scene plan that downstream storyboard-table generation uses as a production contract. The fix must replace the placeholder-level chapter-001 plan with a deterministic Toonflow-grade Daojie plan and add validation so the regression cannot silently return.

## Data Flow

```text
Daojie script + canonical 43 shots
  -> build_daojie_chapter001_workflow.py
  -> raw agentWorkData["directorPlan"] as <scriptPlan> markdown
  -> structured state["scriptPlans"] ScriptPlan object
  -> buildStudioFlowData() raw preview preference
  -> formatScriptPlanContext() downstream storyboard table context
  -> automate-daojie-chapter001-video.mjs report validation
```

## Boundaries

- Source of truth for this task is `Library/build_daojie_chapter001_workflow.py`.
- The change is chapter-001-specific because the canonical shots and Daojie assets are chapter-001-specific.
- `agentWorkData["directorPlan"]` stores the full raw markdown in `<scriptPlan>` tags for preview and Toonflow-style traceability.
- `state["scriptPlans"]` stores a structured projection for frontend workflows and downstream context builders.
- The automation wrapper validates metrics from the generated report and must fail on short director plans.
- The UI markdown preview should keep using raw `agentWorkData` when present; no new persistent UI model field is required.

## Raw Plan Contract

The raw plan should be markdown inside `<scriptPlan>...</scriptPlan>` and contain:

- `## ① 主题立意与叙事核心`
- `## ② 视觉风格与画面基调`
- `## ③ 叙事结构与节奏规划`
- `## ④ 分场景情绪与画面意图`
- `### Sc 1-1 金水河码头 傍晚/外`
- `### Sc 1-2 悦来客栈 夜/内`
- `### Sc 1-3 金水塾馆 夜/内`
- `### Sc 1-4 悦来客栈斗室 深夜/内`
- `### Sc 1-5 悦来客栈窗前/金水河 深夜/外`
- `## ⑤ 声音方向`
- `## ⑥ 转场与视觉连续性`
- `### ⑦ 衍生资产预划清单`

The `⑦` derived asset section may remain a `###` section so existing derived-asset parsing keeps working. It is not part of the six rich Toonflow-grade planning sections.

Current MYStudio `scriptDraft` has four explicit scene heads (`1-1` through `1-4`). The fifth director-planning scene is a tail-hook split from the current `1-4 悦来客栈斗室 深夜/内外` ending, separating interior recognition from exterior spiritual-boat pressure. This follows the Toonflow EP01 planning shape without adding storyboards or changing script text.

## Structured Plan Contract

The structured plan written to `state["scriptPlans"]` should include:

- `theme`: compact but detailed theme and emotional spine from section `①`.
- `visualStyle`: Daojie visual plan from section `②`, using existing water-ink vocabulary.
- `narrativeRhythm`: section `③`, including paragraph rhythm and turning points.
- `sceneIntents`: five entries:
  - `sceneId`
  - `emotion`
  - `shotIntent`
  - `spatial`
- `soundDirection`: section `⑤`.
- `transitions`: section `⑥`.
- `derivedAssetPlan`: existing `DERIVED_ASSET_PLAN`.

## Audit Contract

Add a Python-side audit helper so metrics are computed from the generated raw and structured plan once:

- `directorPlanChars`
- `directorPlanChineseChars`
- `directorPlanH2Sections`
- `directorPlanSceneSections`
- `directorPlanBulletCount`
- `directorPlanRequiredSectionsPresent`
- `directorPlanRequiredSceneSectionsPresent`
- `directorPlanStructuredSceneIntents`
- `directorPlanStructuredSceneIntentsComplete`
- `directorPlanHasDerivedAssetSection`

The JSON report should include these fields. The Node automation wrapper should validate them after loading the generated report.

## Parser Compatibility

`parseDirectorPlan()` already supports:

- compact Toonflow-style sections: `分场汇总表`, `逐场注意事项`, `场间过渡`
- legacy `①-⑦` sections, but currently sets `sceneIntents: []` for rich `④` plans.

To preserve rich plans edited in the UI or pasted from Toonflow:

- Keep the existing compact path unchanged.
- Add a parser for `### Sc ...` blocks inside section `④`.
- Populate `sceneIntents` from each `### Sc` block:
  - `sceneId`: heading prefix, e.g. `Sc 1-1`
  - `emotion`: the `情绪目标` or `氛围方向` bullet content
  - `shotIntent`: the `镜头意图` bullet list or first intent paragraph
  - `spatial`: `空间叙事` plus `距离感设计` content
- Keep `stripLightingTerms()` behavior in structured fields to avoid breaking existing light-separation tests. Raw `agentWorkData` preview may still contain the full plan generated by the Python automation.

## Validation Strategy

Fast validation is enough before full media generation:

- Python compile proves the generator remains syntactically valid.
- A focused Python probe imports the generator and checks raw/structured/audit metrics without invoking TTS or image generation.
- Focused Vitest coverage checks:
  - build script contains new director-plan audit/report gates.
  - parser preserves `### Sc` scene intents.
  - flow data still prefers raw `directorPlan` and formats structured fallback.
  - downstream context still includes scene intents and derived assets.
- Typecheck and lint catch cross-layer TypeScript regressions.

Full media generation is optional unless the user explicitly asks for real generation in this task. If run, it must still prove the existing 43 storyboard, real TTS, real image workflow, and final MP4 stream gates.

## Trade-Offs

- Deterministic chapter-001 plan vs model-generated plan:
  - Deterministic is chosen for repeatable tests and project writeback.
  - Model-generated manual flow remains available for interactive use.
- Rich raw markdown vs new schema:
  - Rich markdown is chosen because existing preview/downstream flow data already expects raw Toonflow-style text.
  - Structured `ScriptPlan` remains a projection, not a replacement for the raw plan.
- Parser enhancement vs no parser change:
  - Parser enhancement is required because otherwise UI edits or re-parsing rich `①-⑥` content would drop all five scene intents.
