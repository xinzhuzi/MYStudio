# Design: docs 清理

## 边界

- 可修改范围：`docs/` 与本任务目录。
- 不修改应用源码、测试、配置和其他任务资料。
- 不执行 git / worktree、批量删除、清空或 Trellis 归档。

## 分类方法

每个文件按以下证据顺序判断：

1. 是否为系统、临时或备份文件。
2. 是否被文档索引覆盖。
3. 是否有仓库内 Markdown 反向链接。
4. 是否被未完成任务作为上下文或验收产物引用。
5. 是否与权威文档完全重复或职责重叠。
6. 是否包含当前仍有效、但权威文档缺失的内容。

没有完整证据链的文件默认保留。

## 最小变更

1. 删除系统元数据 `docs/.DS_Store`。
2. 把孤立 GPT 图片适配说明中的当前有效内容并入 `docs/API_PROVIDER_MODEL_TEST_REFERENCE.md`：
   - GPT 图片模型测试走真实 `sdkGenerateImage()` 链路。
   - 默认使用 `openai-size` 与 `/v1/images/generations`。
   - 不支持标准 `size` 时才走供应商扩展兜底。
   - 记录当前尺寸限制和常见错误边界。
3. 删除已完成合并的 `docs/融合/GPT图片生成标准适配说明.md`。
4. 在 `docs/融合/README.md` 增加“当前审计资料”，索引两份仍被任务使用的 Toonflow 文档。

## 一致性与兼容性

- 不改变应用行为，只修正文档事实和索引结构。
- 不改已有文档路径，避免破坏外部收藏或任务引用。
- 删除 GPT 孤立说明前，先确认权威文档已覆盖其仍有效的操作信息。
- 两份 Toonflow 文档保留原路径，避免破坏未完成任务上下文。

## 风险与回退

- 风险：误删仍被使用的审计资料。控制：全树反向引用和任务状态核验已完成，引用中的文件不删。
- 风险：合并后遗漏 GPT 图片排错信息。控制：删除前逐项对照标准模板、尺寸、兜底和错误四类内容。
- 风险：索引新增后出现坏链。控制：实施后运行全量 Markdown 相对链接检查。
- 如果合并验证失败，则不删除 GPT 孤立说明，保留原文件并回到规划。
